В папке /docker-image
существует два файла .yml для создания образа запуска проекта:

1 docker-compose-development.yml
Создаст образ для дальнейшей  разработки
(При изменении файлов через IDE в папке /news-site-docker-image, будут изменятся и файлы в контейнерах)
Будет использоваться привязка томов.

2 docker-compose-public.yml
Второй классический файл создаст образ с статичными контейнерами (скачает с репозитория hub.docker.com),
без возможности изменения файлов проекта.
Нет привязки томов.
